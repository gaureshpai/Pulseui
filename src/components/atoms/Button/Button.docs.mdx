The Button component is the foundation for all interactive actions in your application. It provides a consistent, accessible, and flexible interface for user interactions.

## Props

The Button component accepts the following props:

| Prop                 | Type                                                                   | Default       | Description                       |
| -------------------- | ---------------------------------------------------------------------- | ------------- | --------------------------------- |
| `children`           | `ReactNode`                                                            | -             | Button content                    |
| `variant`            | `'filled' \| 'subtle' \| 'light' \| 'outline' \| 'white' \| 'default'` | `'default'`   | Button variant style              |
| `size`               | `'xs' \| 'sm' \| 'md' \| 'lg' \| 'xl'`                                 | `'md'`        | Button size                       |
| `state`              | `'default' \| 'hover' \| 'disabled'`                                   | `'default'`   | Button state                      |
| `leftIcon`           | `boolean`                                                              | `false`       | Show left icon                    |
| `rightIcon`          | `boolean`                                                              | `false`       | Show right icon                   |
| `leftIconComponent`  | `SvgIconComponent`                                                     | `ArrowUpward` | Custom left icon component        |
| `rightIconComponent` | `SvgIconComponent`                                                     | `ArrowUpward` | Custom right icon component       |
| `compact`            | `boolean`                                                              | `false`       | Compact mode with reduced padding |
| `justify`            | `'center' \| 'space-between'`                                          | `'center'`    | Content justification             |
| `onClick`            | `(event: MouseEvent) => void`                                          | -             | Click handler                     |
| `disabled`           | `boolean`                                                              | `false`       | Disabled state                    |
| `type`               | `'button' \| 'submit' \| 'reset'`                                      | `'button'`    | Button type                       |
| `className`          | `string`                                                               | `''`          | Additional CSS classes            |

## Variants

The Button component supports 6 different visual variants to match different use cases and design contexts.

| Variant     | Use Case                                       | Example                                                 |
| ----------- | ---------------------------------------------- | ------------------------------------------------------- |
| **Filled**  | Primary actions, main CTAs                     | `<Button variant="filled">Primary Action</Button>`      |
| **Subtle**  | Secondary actions, less prominent              | `<Button variant="subtle">Secondary Action</Button>`    |
| **Light**   | Tertiary actions, minimal impact               | `<Button variant="light">Tertiary Action</Button>`      |
| **Outline** | Alternative to filled, secondary but important | `<Button variant="outline">Alternative Action</Button>` |
| **White**   | Colored backgrounds, cards                     | `<Button variant="white">Card Action</Button>`          |
| **Default** | Fallback, neutral actions                      | `<Button variant="default">Neutral Action</Button>`     |

## Sizes

The Button component supports 5 different sizes, each with appropriate padding, font size, and icon scaling.

| Size | Padding                                | Font Size             | Min Height | Icon Size | Use Case                      |
| ---- | -------------------------------------- | --------------------- | ---------- | --------- | ----------------------------- |
| xs   | `var(--spacing-xs) var(--spacing-sm)`  | `var(--font-size-xs)` | 24px       | xs (16px) | Dense layouts, toolbars       |
| sm   | `var(--spacing-sm) var(--spacing-md)`  | `var(--font-size-sm)` | 32px       | sm (20px) | Inline elements, compact UI   |
| md   | `var(--spacing-md) var(--spacing-lg)`  | `var(--font-size-md)` | 40px       | md (24px) | Standard buttons, forms       |
| lg   | `var(--spacing-lg) var(--spacing-xl)`  | `var(--font-size-lg)` | 48px       | lg (28px) | Prominent actions, CTAs       |
| xl   | `var(--spacing-xl) var(--spacing-xxl)` | `var(--font-size-xl)` | 56px       | lg (28px) | Hero sections, large displays |

## Best Practices

### ✅ Do

| Practice                                            | Description                                      |
| --------------------------------------------------- | ------------------------------------------------ |
| **Use `filled` for primary actions**                | Main action on a page or in a section            |
| **Use `outline` or `subtle` for secondary actions** | Less prominent but still important actions       |
| **Use `light` for tertiary actions**                | Minimal visual impact actions                    |
| **Include descriptive text with icons**             | Always provide text labels with icons            |
| **Use appropriate sizes for context**               | Match button size to its importance and context  |
| **Provide clear click handlers**                    | Ensure onClick handlers are properly defined     |
| **Test keyboard navigation**                        | Verify all buttons work with keyboard navigation |

### ❌ Don't

| Practice                                         | Description                                    |
| ------------------------------------------------ | ---------------------------------------------- |
| **Use too many button variants on one page**     | Stick to 2-3 variants maximum per page         |
| **Make buttons too small for touch interaction** | Ensure minimum 44px touch target               |
| **Use icons without descriptive text**           | Always provide text labels for accessibility   |
| **Override design tokens unnecessarily**         | Use the provided design tokens for consistency |
| **Use buttons for navigation**                   | Use links instead for navigation               |
| **Forget to handle loading states**              | Provide loading states for async actions       |
